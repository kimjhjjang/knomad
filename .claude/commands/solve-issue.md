# GitHub 이슈 해결 계획 커스텀 커맨드

## 사용 방법

GitHub 이슈 번호를 입력하면, 해당 이슈를 분석하고 해결을 위한 상세 계획을 수립합니다.

## 프롬프트

**이슈 번호:** #$ARGUMENTS

---

### 1단계: 이슈 정보 수집

다음 명령어로 GitHub 이슈 정보를 가져와주세요:

```bash
gh issue view $ARGUMENTS --json title,body,labels,assignees,state,comments,createdAt,author
```

---

### 2단계: 이슈 분석

가져온 이슈 정보를 바탕으로 다음을 분석해주세요:

#### 📋 이슈 요약

| 항목 | 내용 |
|------|------|
| 제목 | [이슈 제목] |
| 유형 | bug / feature / enhancement / etc |
| 상태 | open / closed |
| 작성자 | [작성자] |
| 라벨 | [라벨 목록] |
| 생성일 | [날짜] |

#### 🎯 핵심 요구사항

- [이슈에서 요구하는 핵심 사항 1]
- [이슈에서 요구하는 핵심 사항 2]
- [이슈에서 요구하는 핵심 사항 3]

#### ❓ 불명확한 점

- [추가 확인이 필요한 사항이 있다면 나열]

---

### 3단계: 코드베이스 분석

이슈 해결에 필요한 현재 코드 상태를 파악해주세요:

#### 🔍 관련 파일 탐색

- 이슈와 관련된 파일들을 찾아서 나열
- 각 파일의 현재 상태와 역할 설명
- 수정이 필요한 부분 식별

#### 📊 영향 범위

- 직접 수정이 필요한 파일
- 간접적으로 영향받는 파일
- 테스트가 필요한 영역

---

### 4단계: 해결 계획 수립

#### 🛠️ 구현 계획

다음 형식으로 상세한 실행 계획을 작성해주세요:

```markdown
## 해결 계획: [이슈 제목]

### Phase 1: [단계명]

**목표**: [이 단계에서 달성할 것]

- [ ] **Task 1.1**: [작업명]
  - 파일: `[파일 경로]`
  - 작업 내용: [구체적인 수정 내용]
  - 예상 변경:
    ```typescript
    // Before
    [기존 코드 스니펫]

    // After
    [변경될 코드 스니펫]
    ```

- [ ] **Task 1.2**: [작업명]
  - 파일: `[파일 경로]`
  - 작업 내용: [구체적인 수정 내용]

**검증**: [이 Phase 완료 확인 방법]

---

### Phase 2: [단계명]
(동일한 형식으로 계속...)

---

### 검증 계획

- [ ] 기능 테스트: [테스트 방법]
- [ ] 빌드 확인: `npm run build`
- [ ] 회귀 테스트: [영향받는 다른 기능 확인]

### 예상 소요 시간

| Phase | 예상 시간 |
|-------|----------|
| Phase 1 | X 시간 |
| Phase 2 | X 시간 |
| 검증 | X 시간 |
| **총계** | **X 시간** |

### 위험 요소

- [잠재적 문제점과 대응 방안]

### 대안 접근법

- [다른 해결 방법이 있다면 나열]
```

---

### 5단계: 사용자 검토 요청

계획 작성 완료 후, 다음 질문으로 사용자 검토를 요청해주세요:

```
## 📝 검토 요청

위 계획을 검토해주세요:

1. **범위**: 이슈의 요구사항을 모두 포함했나요?
2. **접근법**: 제안한 해결 방법이 적절한가요?
3. **우선순위**: 작업 순서가 합리적인가요?
4. **누락**: 빠진 부분이나 추가할 내용이 있나요?

---

**다음 중 선택해주세요:**

- ✅ **승인**: 계획대로 진행
- 🔄 **수정 요청**: [수정이 필요한 부분 알려주세요]
- ❌ **재작성**: 다른 접근법으로 다시 계획
- ❓ **질문**: [추가 정보가 필요한 경우]
```

---

### 6단계: 승인 후 실행

사용자가 승인하면:

1. 계획에 따라 순차적으로 작업 실행
2. 각 Task 완료 시 체크 표시
3. Phase 완료 시 중간 검증 수행
4. 전체 완료 후 최종 검증 및 결과 보고

---

### 출력 원칙

1. **구체적으로**: 파일 경로, 코드 스니펫, 명령어 포함
2. **단계적으로**: 작은 단위로 분해하여 추적 가능하게
3. **검증 가능하게**: 각 단계마다 완료 확인 방법 명시
4. **대화형으로**: 사용자 피드백을 반영할 수 있는 구조
