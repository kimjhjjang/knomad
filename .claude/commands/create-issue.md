# GitHub 이슈 생성 커스텀 커맨드

## 사용 방법

이슈로 등록하고 싶은 내용을 입력하면, GitHub 이슈 형식에 맞게 구조화된 이슈를 생성합니다.

## 프롬프트

다음 내용을 기반으로 GitHub 이슈를 작성해주세요:

**이슈 내용:** $ARGUMENTS

---

### 분석 및 작성 지침

1. **이슈 유형 판별**: 입력 내용을 분석하여 적절한 이슈 유형 결정
   - `bug`: 버그 리포트
   - `feature`: 새로운 기능 요청
   - `enhancement`: 기존 기능 개선
   - `docs`: 문서 관련
   - `refactor`: 리팩토링
   - `chore`: 유지보수 작업

2. **우선순위 판단**
   - `priority: critical`: 즉시 해결 필요
   - `priority: high`: 빠른 시일 내 해결
   - `priority: medium`: 일반적인 우선순위
   - `priority: low`: 여유 있을 때 처리

3. **영향 범위 분석**
   - 영향받는 컴포넌트/페이지 식별
   - 관련 파일 경로 추정

---

### 출력 형식

다음 형식으로 GitHub 이슈를 작성해주세요:

```markdown
## 이슈 제목

[유형] 간결하고 명확한 제목 (50자 이내)

---

## 이슈 본문

### 📋 설명

[이슈에 대한 상세 설명. 무엇을, 왜 해야 하는지 명확하게 작성]

### 🎯 목표

- [ ] 달성해야 할 목표 1
- [ ] 달성해야 할 목표 2
- [ ] 달성해야 할 목표 3

### 📁 관련 파일

- `경로/파일명.tsx` - 설명
- `경로/파일명.ts` - 설명

### 🔗 관련 이슈/PR

- 관련 이슈가 있다면 #번호로 연결

### 💡 구현 힌트 (선택)

[구현 방향이나 참고할 만한 내용이 있다면 작성]

### ✅ 완료 조건

- [ ] 조건 1을 만족하면 완료
- [ ] 조건 2를 만족하면 완료

---

## 라벨

- `type: [bug/feature/enhancement/docs/refactor/chore]`
- `priority: [critical/high/medium/low]`
- `area: [frontend/backend/infra/design]`

---

## gh CLI 명령어

아래 명령어로 이슈를 바로 생성할 수 있습니다:

\`\`\`bash
gh issue create --title "[유형] 제목" --body "본문 내용" --label "type:feature,priority:medium"
\`\`\`
```

### 이슈 유형별 추가 섹션

**버그 리포트인 경우 추가:**

```markdown
### 🐛 버그 상세

**재현 단계:**
1. '...'로 이동
2. '...'를 클릭
3. 에러 발생

**예상 동작:**
[정상적으로 동작해야 하는 방식]

**실제 동작:**
[현재 발생하는 문제]

**환경:**
- 브라우저:
- OS:
- 버전:
```

**기능 요청인 경우 추가:**

```markdown
### ✨ 기능 상세

**사용자 스토리:**
[사용자]로서, [목적]을 위해, [기능]을 원한다.

**수용 기준:**
- [ ] 기준 1
- [ ] 기준 2

**디자인/와이어프레임:**
[있다면 첨부 또는 설명]
```

---

### 작성 원칙

1. **제목은 명확하게**: 무엇을 해야 하는지 한눈에 파악 가능
2. **본문은 구체적으로**: 배경, 목적, 완료 조건 포함
3. **체크리스트 활용**: 진행 상황 추적 가능하게
4. **라벨 권장**: 필터링과 우선순위 관리에 활용
5. **CLI 명령어 제공**: 바로 실행 가능한 명령어 포함
