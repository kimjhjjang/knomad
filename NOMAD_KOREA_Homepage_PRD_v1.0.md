## 1. 문제 정의 (Problem Definition)

### 1.1 배경

대한민국에서 디지털 노마드로 생활하고자 하는 사람들이 증가하고 있지만, 도시별 생활 조건(인터넷 속도, 생활비, 코워킹 스페이스, 교통 등)을 체계적으로 비교하고 평가할 수 있는 전문 플랫폼이 존재하지 않습니다.

### 1.2 현재 문제점

| # | 문제 | 상세 설명 |
|---|------|-----------|
| 1 | **정보 분산** | 도시별 노마드 관련 정보가 블로그, 카페, SNS 등에 산재되어 있어 체계적인 비교가 어려움 |
| 2 | **신뢰도 부족** | 개인 후기 위주의 정보로 객관적이고 다차원적인 평가 데이터가 부재 |
| 3 | **비교 불가** | 여러 도시의 생활 조건을 동일한 기준으로 비교할 수 있는 도구가 없음 |
| 4 | **실시간성 부족** | 기존 정보는 오래된 경우가 많아 현재 상황을 정확히 반영하지 못함 |

### 1.3 기회

한국 디지털 노마드 커뮤니티의 성장과 함께, 도시별 생활 정보를 카드 형태로 직관적으로 제공하고 사용자 평가를 통해 신뢰도 높은 정보를 축적하는 플랫폼은 시장에서 독보적인 포지션을 가질 수 있습니다.

## 2. 타겟 사용자 (Target Users)

### 2.1 주요 타겟

디지털 노마드로 살고 싶거나, 현재 노마드 생활을 하고 있는 사람들을 타겟팅합니다.

### 2.2 사용자 특성

| 특성 항목 | 상세 설명 |
|-----------|-----------|
| **디지털 리터러시** | 컴퓨터 및 디지털 도구 활용에 능숙하며, 새로운 웹 서비스 적응이 빠름 |
| **SNS 활동** | SNS에서 활발히 활동하며 정보 공유와 커뮤니티 참여에 적극적 |
| **직업군** | 소프트웨어 개발, 글쓰기, 마케팅, 디자인 등 원격 근무 가능한 직종 종사자 |
| **업무 환경** | 특정 사무 공간에 구애받지 않으며, 카페나 코워킹 스페이스에서 업무 가능 |
| **연령대** | 주로 20~40대의 MZ세대 |
| **라이프스타일** | 새로운 환경과 경험을 추구하며, 유연한 삶의 방식을 선호 |
| **정보 탐색** | 이동 전 충분한 사전 조사를 하며, 다른 노마드의 후기와 평가를 중요시함 |

### 2.3 사용자 페르소나

> **페르소나: 김민수 (32세, 프리랜서 개발자)**
>
> 서울에서 2년간 프리랜서 개발자로 일하고 있으며, 높은 월세와 빠른 도시 생활에 지쳐 지방 도시로의 이동을 고려 중입니다. 인터넷 속도가 안정적이고, 카페나 코워킹 스페이스가 있는 도시를 찾고 있으며, 월 생활비 150만원 이하의 도시를 희망합니다.
>
> **핵심 니즈:** 도시별 인터넷 속도, 생활비, 코워킹 인프라를 한눈에 비교하고, 실제 거주한 노마드들의 생생한 후기를 확인하고 싶음

## 3. 제안 해결책 (Proposed Solution)

### 3.1 솔루션 개요

대한민국 주요 도시들의 노마드 생활 조건을 카드 형태로 직관적으로 제공하고, 사용자들이 다차원적으로 평가할 수 있는 웹 플랫폼 **'NOMAD KOREA'**를 구축합니다.

### 3.2 핵심 가치 제안 (Value Proposition)

| 가치 | 설명 |
|------|------|
| **직관적 정보 탐색** | 카드 기반 UI로 도시별 핵심 정보를 한눈에 파악 가능 |
| **다차원 평가 시스템** | 9개 카테고리별 세분화된 평가로 신뢰도 높은 데이터 축적 |
| **커뮤니티 기반 정보** | 실제 노마드들의 리뷰와 평가로 살아있는 정보 제공 |
| **맞춤형 추천** | 필터, 정렬, 퀴즈 기능으로 개인에 맞는 도시 추천 |

### 3.3 범위 (Scope)

본 PRD는 **홈페이지(Homepage)** 에 한정하여 기능 요구사항을 정의합니다. 도시 상세 페이지, 비교 페이지, 커뮤니티 등 서브 페이지에 대한 기획은 별도 PRD에서 다룹니다.

## 10. 제약 사항 및 참고

### 10.1 범위 제한

본 PRD는 홈페이지에 한정됩니다. 다음 항목은 별도 PRD에서 다룹니다:

3. 사용자 계정 시스템 (회원가입, 소셜 로그인, 프로필 관리)
4. 사용자 리뷰 작성 시스템 (카테고리별 평가, 텍스트 리뷰, 사진 첨부)

6. 관리자 페이지 (도시 데이터 관리, 리뷰 모더레이션)

### 10.2 기술적 제약

초기 MVP에서는 도시 데이터를 정적 JSON 또는 Supabase DB에서 관리하며, 사용자 생성 콘텐츠(리뷰)는 2단계에서 도입합니다. 인기 도시 순위는 초기에는 수동 관리하며, 리뷰 시스템 도입 후 자동 집계로 전환합니다.
